---
name: plan
objective: Time-based organization into daily priorities and milestones toward a goal.
format: structured timeline with milestones and daily breakdowns
layout:
  - Goal
  - Milestones
  - Daily Priorities
  - Dependencies and Risks
  - Definition of Done
context: Use when source material describes a goal or project that needs to be broken into time-ordered actions. Good for project planning, sprint organization, personal goal-setting, and roadmap creation.
scope: Every item must be actionable and time-bound. Milestones must be concrete checkpoints, not vague phases. Dependencies must be explicit. The plan must be executable, not aspirational.
reference: Use instruction for step-by-step procedural breakdowns. Use manager persona for execution focus.
---
Produce an actionable time-based plan organized around a clear goal.

Required output sections:
- **# Goal** — the specific, measurable outcome this plan is working toward; one to two sentences
- **# Milestones** — the major checkpoints between now and the goal; each milestone has a name, a target date or timeframe, and a one-sentence description of what done looks like
- **# Daily Priorities** — a day-by-day or week-by-week breakdown of specific actions; each entry states what to do, not just what to think about; format as a list under each time period
- **# Dependencies and Risks** — things that must happen first or that could derail the plan; include mitigation for each risk
- **# Definition of Done** — explicit criteria for knowing the goal has been achieved

Every item in Daily Priorities must be a concrete action someone can complete. Vague tasks like "work on X" or "think about Y" are not allowed — replace them with specific deliverables.
